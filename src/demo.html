Introduction

I've been building different kind of website for quite a long time now, and i've seen myself having similar routine. Thinking about form and button, firefox and IE8, make sure grunt handle of every task possible.

This is what i use as a very basic boilerplate for all my project. Simple logicisare already set in place (grid, media query, UI element) but the code is extremly simple & maleable. Build, server and templating task are present and let me start a new project instantly.

This is a concept-free boilerplate, good for webapp, blog, forum or any other kind of website. That's one of the reason why the spacing is separated in is own file. (webapp think about spacing differently that classical website).

Another main concept is the use of rem (main.css) with a fallback to px (main-no-rem.css) for non-supportive browser. Using rem system (with help of the rem build-in mixin) assure a consistant layout around all the site. try to change the body font-size and you'll see that everything is responding to that unique value

Use the build in font-size mixin:

The mixin ask one argument (font-size) but a second argument can be specified (line-height) font-size & line-height

Example :

// Sass
.paragraphe-bigger {
  @include font-size(1.2);
  // font-size
}
// main.css
.default-line-height {
  font-size: 1.2rem;
  line-height: 1.92rem;

// main-no-rem.css
.default-line-height {
  font-size: 19.2px;
  line-height: 26.8px;

// sass
.same-line-height {

.same-line-height {

  line-height: 1.2rem;

// main-no-rem.css
.same-line-height {

To change your grid system, change those line in the _variable.scss file:

/* Grid settings */
$grid-width:             1080px;
$gutter-width:           12px;
$columns-number:         12;
$columns-number-string:  "one", "two", "three", "four", "five", "six", "seven", "eight", "nine", "ten", "eleven", "twelve";

You can also use the sass build in mixin for your layout:

@include grid-row();
@include grid-columns($columns:3, $offset:2, $mobile:6);

Variable The first thing you should look at

Every element is linked somehow to that file, it contain default font size & family, grid width, columns number or even columns name. A good practice is to be sure you've got control of your css, and that's by blab lballb a soe consistency toe bla.

Start by playing around your base font size and the line height (default to golden ratio).

$base-font-size:	16px;
$line-height: 	1.61803398875; // Golden ratio

Media query - helper & mixin

There is two breakpoint, $break-small and $break-break-large, they default to 760px and 1080 (the grid width). Helper class contain hide-for-small, hide-for-mediuml and hide-for-large.

The content above will disapear on small screen

The content above will disapear on medium screen

The content above will disapear on large screen
hide on large

Change breakpoint settings by editing the file _variable.scss:

$break-small: 760px;
$break-large: $grid-width; //default: 1080px;

You can use the build in media query mixin:

@include respond-to(handhelds) { ... }
@include respond-to(medium-screens) { ... }
@include respond-to(large-screens) { ... }
@media only screen and (min-width: 1200px) {
  .hide-for-large {
    display: none;
  }
}

Spacing To infinite and beyond!

Tips Good pratice is the key

1. rem that mofo

Since all the css is based around rem, I strongly advice to use the rem mixin when it comes to margin or padding calculation. This will ensure a uniform spacing on all the website and make that the px fallback will be present. The mixin ask two argument, property & rem value

Example :

// sass
#navigation {
  @include rem(margin-bottom, 2);
}	

// main.css
#navigation {
  margin-bottom: 1rem;
}	

// main-no-rem.css
#navigation {
  margin-bottom: 16px;
}
